# ActionCap

[English](README.md)

浏览器活动录制工具，适用于 Edge / Chrome。录制浏览过程中的页面网络请求与用户操作，内置基于 rrweb 的会话回放功能。

## 功能特性

**三种录制范围**
- **当前 Tab** — 仅录制单个标签页的活动。
- **跨 Tab** — 录制当前窗口内所有标签页。
- **所有窗口** — 录制所有浏览器窗口的所有标签页。

**完整网络捕获**
- 通过 DevTools Protocol 捕获请求/响应的 Headers 和 Body。
- 支持 JSON、HTML、XML、表单数据等文本类型内容。
- 大体积响应自动截断，二进制内容自动过滤。

**用户操作追踪**
- 记录点击、双击、右键、键盘输入、表单提交、滚动、焦点变化、页面跳转等操作。
- 为每个元素生成稳定的选择器（`data-testid` > `id` > `name` > `aria-label` > CSS 路径）。

**会话回放**
- 使用 rrweb 实现录制会话的可视化回放。
- 完整的 DOM 快照与增量变更追踪。

**时间线与分析**
- 统一时间线，按时间顺序展示网络事件与用户操作。
- 按类型筛选：全部 / 操作 / 网络 / 错误。
- 支持搜索操作、请求、URL。
- 详情面板提供格式化的请求/响应内容查看器。

**会话管理**
- 支持重命名、删除、导入和导出会话。
- 会话以 `.bxdac` 文件（JSON 格式）导出。
- 所有数据存储在本地 IndexedDB 中，不会上传到任何服务器。

## 安装

### Edge 应用商店（推荐）

从 [Microsoft Edge 应用商店](https://microsoftedge.microsoft.com/addons/detail/actioncap/dckc1jp4j3) 直接安装。

### 手动安装

1. 从 [Releases](https://github.com/skye-z/ActionCap/releases) 下载最新的 `.zip` 文件。
2. 解压文件。
3. 打开 `edge://extensions` 或 `chrome://extensions`。
4. 开启 **开发者模式**。
5. 点击 **加载已解压的扩展** 并选择解压后的文件夹。

## 使用方法

1. 点击浏览器工具栏中的 ActionCap 图标。
2. 选择录制范围（当前 Tab / 跨 Tab / 所有窗口）。
3. 点击 **开始录制** — 浏览器会显示调试提示栏，这是正常现象。
4. 正常浏览网页，所有网络请求和用户操作都会被捕获。
5. 再次点击 ActionCap 图标，按下 **结束录制**。
6. 点击 **查看会话** 打开结果页面，查看时间线、检查请求详情或回放会话。

## 常见问题

**为什么录制时浏览器顶部会出现"调试"提示栏？**
ActionCap 使用 `debugger` 权限通过 Chrome DevTools Protocol 捕获网络流量。浏览器在任何扩展激活调试器时都会显示此提示栏，这是浏览器的安全机制。录制停止后提示栏会自动消失。

**能录制 `chrome://` 或 `edge://` 页面吗？**
不能。浏览器内部页面不允许扩展注入脚本，ActionCap 只能录制普通网页。

**我的数据存储在哪里？**
所有录制数据存储在浏览器本地的 IndexedDB 中。ActionCap 不会将任何数据上传到远程服务器。数据只有在你手动导出会话时才会离开设备。

**导出的会话文件是否包含敏感信息？**
是的。导出文件可能包含录制过程中捕获的密码、Token、Cookie 和其他敏感数据。请不要将导出文件分享给不信任的人。详情请参阅 [隐私政策](https://github.com/skye-z/ActionCap/blob/main/docs/PRIVACY_POLICY.md)。

**为什么某些响应内容缺失或被截断？**
过大的响应体会自动截断以节省存储空间。二进制内容（图片、字体等）不会被捕获。

**ActionCap 会影响页面性能吗？**
正常录制时影响极小。但在请求量极大的页面上，内存占用可能会略有增加。
